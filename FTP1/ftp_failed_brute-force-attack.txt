1. Executive Summary

    Incident Type: Brute Force Attack

    Status: Blocked / Failed (No unauthorized access gained)

    Severity: Medium (High frequency of attempts)

    Detection Source: Network Traffic Analysis (PCAP)

2. Attacker & Target Information

    Source IP: 192.168.0.20

    Target Server IP: 192.168.0.21

    Target Port: 21 (FTP)

    Tool Identified: Likely Hydra or an automated script (indicated by high request frequency).

3. Analysis & Findings

Based on the analysis of the provided PCAP file, the following observations were made:

    Observation: A significant volume of FTP connection requests originated from the source IP 192.168.0.20 targeting the server at 192.168.0.21.

    Traffic Pattern: The traffic exhibits a classic Dictionary Attack pattern, iterating through multiple common usernames and passwords in a very short duration.

    Evidence of Failure: * 100% of server responses returned status code 530 Login incorrect.

        The Wireshark filter ftp.response.code == 230 returned zero results, confirming that no successful login occurred.

        Frequent RST (Reset) and FIN (Finish) flags indicate sessions were terminated immediately following each failed attempt.

4. Technical Artifacts

    Wireshark Filter Applied: ip.addr == 192.168.0.20 && ip.addr == 192.168.0.21 && ftp

    Log Correlations:

        Linux: Entries in /var/log/auth.log on 192.168.0.21 would reflect FAILED PASSWORD for the targeted users.

        Windows: If the target was Windows-based, Event ID 4625 (An account failed to log on) would be generated thousands of times.

5. Recommendations & Remediation

    Immediate Action: Permanently block Source IP 192.168.0.20 at the Network Firewall or via Linux iptables on the server.

        Command: sudo iptables -A INPUT -s 192.168.0.20 -j DROP

    Long-term Hardening:

        Account Lockout: Implement a policy on 192.168.0.21 to lock accounts after 5 failed attempts.

        Intrusion Prevention: Deploy Fail2Ban to automate the detection and blocking of Brute Force sources.

        Secure Protocol: Decommission legacy FTP on the server and migrate to SFTP (Port 22) or FTPS to encrypt credentials.

6. Analyst Conclusion

    "The incident was a high-volume but unsuccessful attempt to compromise FTP credentials on the server 192.168.0.21. No data exfiltration or system compromise was detected. However, the persistence of the attacker warrants inclusion of this IP in the corporate blacklist."
